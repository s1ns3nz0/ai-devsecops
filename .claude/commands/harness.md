다음 과정에 따라 작업 논의 및 구현을 진행하라:

1. 먼저 `/docs/` 하위 문서들을 전부 읽고 이 프로젝트의 기획, 아키텍처, 설계 의도를 파악한다. 필요시 여러 Explore 에이전트를 병렬로 사용한다.

2. 이번 작업을 구현하기 위해 더 구체화해야 할 점, 기술적으로 논의해야 할 점이 있다면 사용자에게 제시한 후 논의를 이어간다.

3. 사용자가 충분히 논의했다고 판단 후 구현 계획 작성을 지시하면, 구현 계획을 여러 phase로 나뉜 초안으로 작성하고 사용자에게 피드백을 요청한다.

   각 phase는 다음 원칙을 따른다:
   - **자기완결성**: 독립된 Claude 세션이 이 phase 파일 하나만 보고 작업을 완수할 수 있어야 한다.
   - **scope 최소화**: 하나의 phase에서 하나의 레이어/모듈만 다룬다.
   - **AC는 실행 가능한 커맨드**: "~가 동작해야 한다" 같은 서술이 아닌 실제 실행 가능한 검증 커맨드를 포함한다.
   - **주의사항은 구체적으로**: "조심해라" 대신 "X를 하지 마라. 이유: Y" 형식.

4. 사용자가 승인하면 `tasks/` 폴더에 다음 파일들을 생성한다:
   - `tasks/{task-dir}/index.json` — phase 목록 + 상태 관리
   - `tasks/{task-dir}/phase{N}.md` — 각 phase별 상세 작업 지시서

   각 phase 파일은 반드시 이 구조를 따른다:
   ```
   # Phase {N}: {이름}

   ## 사전 준비
   - 읽어야 할 문서 경로
   - 이전 phase에서 생성/수정된 파일 경로

   ## 작업 내용
   {구체적인 구현 지시}

   ## Acceptance Criteria
   {실행 가능한 검증 커맨드}

   ## AC 검증 방법
   AC 커맨드를 실행하라. 통과하면 index.json의 status를 "completed"로 변경.
   수정 3회 시도 후에도 실패하면 "error"로 변경하고 error_message를 기록.
   사용자 개입이 필요하면 "blocked"로 변경하고 blocked_reason을 기록 후 즉시 중단.

   ## 주의사항
   - {하지 말아야 할 것}
   ```

5. `scripts/execute.py`를 실행해서 각 phase를 순차적으로 실행한다.
